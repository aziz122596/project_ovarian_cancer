# Методология

## 1. Источники данных

Использовано 4 независимых протеомных проекта PRIDE:

| Проект | n | Материал | Cancer | Control |
|---|---|---|---|---|
| PXD033741 | 110 | Ткань | 80 | 30 |
| PXD009655 | 50 | Микровезикулы (UtL) | 43 | 7 |
| PXD012998 | 14 | Ткань | 10 | 4 |
| PXD025864 | 6 | Ткань | 3 | 3 |
| **Итого** | **180** | — | **136** | **44** |

## 2. Препроцессинг

Препроцессинг каждого проекта проведён отдельно перед объединением:

1. **Идентификация и квантификация:** MaxQuant v2.4.7.0 + Andromeda search engine, поиск против UniProtKB Human (релиз 2025_01).
2. **Фильтры:** удаление контаминантов, реверсов, идентифицированных только по сайту, FDR < 1%.
3. **Квантификация:** iBAQ.
4. **Логарифмирование:** log2(iBAQ + 1) при необходимости.
5. **Z-score нормализация:** внутри каждого проекта отдельно (для удаления систематических сдвигов).
6. **Импутация пропусков НЕ применялась**

Объединённая матрица: 9 610 белков × 180 образцов.

## 3. Отбор коровых белков

**Определение:** белки, имеющие валидное (не-NaN) значение **во всех 180 образцах одновременно** (coverage = 100%).

**Логика:** если белок надёжно детектируется во всех образцах, значит, он стабильно квантифицируется любым прибором, в любой лаборатории, любым протоколом. Это автоматический фильтр воспроизводимости, не требующий импутации.

**Результат:** 2 701 белок из 9 610 (28%).

## 4. Стратегии интеграции мультицентровых данных

В работе протестированы **4 альтернативные стратегии** обработки гетерогенности данных:

### A) Tissue-only LOPO
- Используется только тканевая подвыборка (130 образцов из 3 проектов).
- Микровезикулы (PXD009655) исключены как биологически отличный матрикс.
- Валидация: Leave-One-Project-Out (обучение на 2 проектах, тест на 1).

### B) Universal markers
- Отбираются белки, статистически значимо изменённые в раке **одновременно в ткани и в EVs**:
  - p < 0.05 в обоих матриксах
  - |t-stat| > 1.5 в обоих
  - Согласованное направление эффекта (sign(t_T) == sign(t_E))
- Получено 194 universal markers.
- Модель обучается на этих белках на всех 4 проектах с LOPO.

### C) Stratified — раздельные модели
- Отдельная модель для tissue (LOPO между 3 проектами).
- Отдельная модель для EVs (внутренняя 5-fold CV на одном проекте).

### D) Material-aware feature engineering
- Material-specific z-score нормализация (внутри tissue / внутри EVs).
- Добавление one-hot признака `IsEV`.
- Общая модель + LOPO на всех 4 проектах.

## 5. Алгоритмы машинного обучения

В каждом подходе тестировались 6 алгоритмов с одинаковыми гиперпараметрами:

- **Logistic Regression L2** (C=1.0, lbfgs, balanced)
- **Logistic Regression L1** (C=1.0, liblinear, balanced) — embedded feature selection
- **Random Forest** (500 trees, max_features='sqrt', balanced)
- **Gradient Boosting** (200 estimators, lr=0.05, max_depth=3)
- **SVM RBF** (C=1.0, balanced)
- **SVM Linear** (C=1.0, balanced)

`class_weight='balanced'` для всех моделей — компенсация дисбаланса cancer/control.

Итого: 4 подхода × 6 алгоритмов = **24 обученные модели**.

## 6. Расширенная валидация

### 6.1 Bootstrap CI
1000 случайных перевыборок с возвращением на pooled-предсказаниях LOPO. Расчёт 95% percentile интервала AUROC и AUPRC.

### 6.2 Permutation test
100 случайных перестановок меток с полной перепрогонкой LOPO. Для ускорения используется feature selection (top-200 по f_classif на train) и уменьшенный RF (100 деревьев). p-value = (число null AUROC ≥ observed + 1) / (n_permutations + 1).

### 6.3 Клинические метрики (Youden J)
- Optimal threshold: argmax(TPR − FPR)
- Sensitivity, Specificity, PPV, NPV
- Confusion matrix

### 6.4 Stability test
1. **Без PXD033741:** LOPO между двумя малыми проектами (PXD012998 + PXD025864, n=20).
2. **Train(small) → Test(big):** обучение на 20 образцах малых, тест на 110 образцах PXD033741.

### 6.5 Hallmark enrichment
Hypergeometric overrepresentation test для топ-100 RF и universal markers против 10 курированных hallmark-категорий онкогенных путей. BH-коррекция на множественные сравнения.
